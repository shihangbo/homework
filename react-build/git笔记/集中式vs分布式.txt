
一、集中式和分布式版本控制系统的区别
1.集中式版本控制系统
  a.版本库是集中存放在中央服务器的，而干活的时候，用自己的电脑，先从中央服务器取得最新的版本，开始干活，干完活了，再把自己的活推送给中央服务器。
  b.必须联网才能工作，对网速，服务器带宽有很大要求
2.分布式版本控制系统
  a.安全性要高很多，因为每个人电脑里都有完整的版本库，每个人修改自己电脑上的版本。
  b.有一个中央库接受所有人的push，也可以随时pull最新版本。

二、git创建版本库
1.初始化一个Git本地仓库，使用git init命令。
2.添加文件到Git本地仓库，分两步：
  a.第一步，使用命令git add <file>，注意，可反复多次使用，添加多个文件；
  b.第二步，使用命令git commit，完成。
3.版本回退
  a.HEAD指向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭，使用命令git reset --hard commit_id。
  b.穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。
  c.要重返未来，用git reflog查看命令历史，以便确定要回到未来的哪个版本。
